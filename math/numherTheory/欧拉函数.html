搞懂欧拉函数：从定义到代码实现

欧拉函数（Euler’s Totient Function, φ\varphiφ）是数论中最重要的基石之一，也是算法竞赛（ACM/ICPC/OI）中的必考点。

在数论和算法竞赛中，我们经常会遇到一个神秘的符号：φ(n)\varphi(n)φ(n)。它不仅是 RSA 加密算法的核心，也是解决很多取模、循环节问题的关键。今天我们就来彻底搞懂欧拉函数及其C++ 代码实现。

1 什么是欧拉函数？

欧拉函数 φ(n)\varphi(n)φ(n) 的定义非常简单：

对于正整数 nnn，φ(n)\varphi(n)φ(n) 表示在小于等于 nnn 的正整数中，与 nnn 互质的数的个数。

注：互质（Coprime）的意思是两个数的最大公约数 gcd⁡(a,b)=1\gcd(a, b) = 1gcd(a,b)=1。

举个栗子： 我们来看 n=6n = 6n=6。 小于等于 6 的数有：1,2,3,4,5,61, 2, 3, 4, 5, 61,2,3,4,5,6。

  • gcd⁡(1,6)=1\gcd(1, 6) = 1gcd(1,6)=1 （互质）✅
  • gcd⁡(2,6)=2\gcd(2, 6) = 2gcd(2,6)=2 （不互质）
  • gcd⁡(3,6)=3\gcd(3, 6) = 3gcd(3,6)=3 （不互质）
  • gcd⁡(4,6)=2\gcd(4, 6) = 2gcd(4,6)=2 （不互质）
  • gcd⁡(5,6)=1\gcd(5, 6) = 1gcd(5,6)=1 （互质）✅
  • gcd⁡(6,6)=6\gcd(6, 6) = 6gcd(6,6)=6 （不互质）

与 6 互质的数只有 1 和 5，共 2 个。所以 φ(6)=2\varphi(6) = 2φ(6)=2。

2 核心计算公式（推导原理）

如果 nnn 很小，我们可以枚举。但如果 nnn 是 10910^9109 级别呢？我们需要通项公式。

根据算术基本定理，任意正整数 nnn 都可以唯一分解为素因子的乘积：

n=p1k1×p2k2×⋯×pmkmn = p_1^{k_1} \times p_2^{k_2} \times \dots \times p_m^{k_m}n=p1k1​​×p2k2​​×⋯×pmkm​​

欧拉函数的通项公式为：

φ(n)=n×(1−1p1)×(1−1p2)×⋯×(1−1pm)\varphi(n) = n \times \left(1 - \frac{1}{p_1}\right) \times \left(1 - \frac{1}{p_2}\right) \times \dots \times \left(1 - \frac{1}{p_m}\right)φ(n)=n×(1−p1​1​)×(1−p2​1​)×⋯×(1−pm​1​)

为什么是这个公式？

利用容斥原理可以通俗理解：

  1. 我们想求 111 到 nnn 中与 nnn 互质的数。
  2. 初始假设所有数都互质，共 nnn 个。
  3. 减去 p1p_1p1​ 的倍数，剩下 n(1−1p1)n(1 - \frac{1}{p_1})n(1−p1​1​)。
  4. 再减去 p2p_2p2​ 的倍数……以此类推。
  5. 注意：欧拉函数只和 nnn 的质因子有关，和质因子的指数（次数）无关。

转换为代码逻辑

在计算机中，浮点数运算（1−1p1 - \frac{1}{p}1−p1​）不仅慢而且有精度误差。我们将公式变形一下，方便整数运算：

n×(1−1p)=n×p−1p=np×(p−1) n \times \left(1 - \frac{1}{p}\right) = n \times \frac{p-1}{p} = \frac{n}{p} \times (p-1) n×(1−p1​)=n×pp−1​=pn​×(p−1)

所以在代码中，对于每一个质因子 ppp，我们只需要执行：res = res / p * (p - 1)。

        typedef long long ll;

// 试除法求欧拉函数
// 输入 n，返回 phi(n)
ll get_phi(ll n) {
    ll res = n; // 初始化结果为 n，对应公式中的 n
    
    // 1. 遍历寻找质因子
    // 我们只需要遍历到 sqrt(n) 即可，这和判断素数的逻辑一样
    for (ll i = 2; i * i <= n; ++i) {
        
        // 2. 如果 i 能整除 n，说明 i 是 n 的一个质因子
        if (n % i == 0) {
            
            // 3. 应用公式：res = res * (1 - 1/i)
            // 变形为整数运算：res = res / i * (i - 1)
            res = res / i * (i - 1);
            
            // 4. 【关键】除尽当前的质因子 i
            // 欧拉函数只跟“有哪些质因子”有关，跟因子的个数无关
            // 例如 12 = 2^2 * 3，我们处理 2 的时候，要把 12 里的两个 2 都除掉，变成 3
            while (n % i == 0) {
                n /= i;
            }
        }
    }
    
    // 5. 处理剩余的那个大质因子
    // 如果循环结束后 n > 1，说明 n 本身还剩下一个大于 sqrt(原n) 的质因子
    // 例如 n = 14 (2 * 7)，循环处理了 2，剩下 7，7 > sqrt(14)，需要再处理一次
    if (n > 1) {
        res = res / n * (n - 1);
    }
    
    return res;
}

4 关键点 Q&A

Q1: 为什么循环只到 n\sqrt{n}n​？ A: 因为一个合数 nnn 最多只有一个大于 n\sqrt{n}n​ 的质因子。我们在循环中处理掉了所有小于等于 n\sqrt{n}n​ 的质因子，最后剩下的 nnn 如果大于 1，那它一定就是那个“最大的质因子”。

Q2: while (n % i == 0) n /= i; 这行代码有什么用？ A: 这是为了保证 iii 必定是质数。 当我们从小到大枚举 iii 时：

  • 当 i=2i=2i=2 时，我们除掉了 nnn 中所有的因子 2。
  • 当枚举到 i=4i=4i=4 时，由于 nnn 已经不包含因子 2 了，所以 nnn 绝不可能被 4 整除。
  • 因此，if (n % i == 0) 成立时，iii 一定是质数。这也避免了我们重复计算同一个质因子。

Q3: 复杂度是多少？ A: 时间复杂度是 O(n)O(\sqrt{n})O(n​)。对于 10910^9109 范围内的数据，计算一次几乎是瞬间完成。

5 总结

求单个数字欧拉函数的步骤：

  1. 初始化答案 res = n。
  2. 从 222 到 n\sqrt{n}n​ 枚举质因子 ppp。
  3. 如果 ppp 是因子：
    • 更新答案：res = res / p * (p - 1)。
    • 将 nnn 中所有的 ppp 除干净。
  4. 如果最后 n>1n > 1n>1，说明还有一个大质因子，再更新一次答案。
