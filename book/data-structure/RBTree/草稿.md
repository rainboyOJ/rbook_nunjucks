证明:


纯粹的旋转操作本身（Pure Rotation）只改变结构，不改变颜色。

1. 双红结点的旋转不会影响平衡.
2. 父p黑,子x红 通过旋转,提升子x红,然后交换p ,x 的颜色,不会影响平衡

以下是按照你的描述绘制的旋转示意图（RBTree 相关，包含旋转前、旋转后和颜色交换后的最终状态）：

一、右旋（x 为 p 的左子节点时）
```
       p(B)               x(R)                       x(B)
      /   \              /   \                      /   \
    x(R)  sr(?)  --->  xl(B)  p(B)      --->      xl(B)  p(R)
   /   \                      / \                        / \
 xl(B) xl(B)                xl(B) sr(?)                xl(B) sr(?)

```
1. Before 旋转前的结构
2. After 旋转后的结构（结构变化，颜色未变）
3. 最终通过颜色交换后的状态

二、左旋（x 为 p 的右子节点时）
```
   p(B)                  x(R)                      x(B)
  /   \                 /    \                    /    \
sl(?)  x(R)   --->     p(B)   xr(B)    --->      p(R)   xr(B)
       / \            /   \                     /   \
     xl(B) xr(B)    sl(?)  xl(B)              sl(?)  xl(B)
```
1. Before 旋转前的结构
2. After 旋转后的结构（结构变化，颜色未变）
3. 最终通过颜色交换后的状态

## 插入与删除的中的递归

核心思想: 自底向上的调整(递归)

- 每一个节点都放一个`调整守卫`(adjust guard)

### 插入

- 基础思想: 上升R节点
- 双R节点的父是调整节点的孩子,`[分情况讨论](4种)`
  - `LL`,`RR`
  - `LR`,`RL`


### 删除

- 基础思想: 上升BB节点
- fixBB, BB节点是孩子,`[分情况讨论]`
- `shiftB`

## fixDB 思考

`c5,c6` 转成 `c7,c8`后 因为双黑节点的父亲是红色,所以提升红色变黑色,不会出现双黑
`c5,c6`转成情况1,就是边界.
