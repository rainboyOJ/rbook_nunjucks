## 一句话算法

位运算就是把整数看成一排 `0/1` 开关，用很少的操作直接检查、打开、关闭或枚举这些开关。

## 问题模型

一个非负整数可以写成二进制：

```text
13 = 1101(2)
```

从右往左，最低位编号为 `0`：

```text
bit:  3 2 1 0
      1 1 0 1
```

第 $k$ 位的权值是 $2^k$。如果第 $k$ 位是 `1`，说明这个整数包含 $2^k$ 这一份贡献。

在算法竞赛中，位运算常用于两类问题：

1. 把一个整数当作普通数字处理，例如判断奇偶、取最低位的 `1`。
2. 把一个整数当作集合状态处理，例如第 $i$ 位表示第 $i$ 个元素是否被选择。

## 核心直觉

位运算的记忆模型很简单：`1` 是开，`0` 是关。

| 操作 | 直觉 | 常用效果 |
|---|---|---|
| `&` | 两边都开才开 | 检查、保留某些位 |
| `|` | 有一边开就开 | 打开某些位 |
| `^` | 不一样才开 | 翻转某些位 |
| `~` | 全部反过来 | 构造反掩码 |
| `<<` | 整体左移 | 乘以 $2^k$ |
| `>>` | 整体右移 | 除以 $2^k$ 向下取整 |

例如要操作第 `k` 位，最常用的掩码是：

```cpp
1ULL << k
```

它只有第 `k` 位是 `1`，其它位都是 `0`。

## 基础操作

下面默认 `k` 从 `0` 开始，`x` 是非负整数。

### 检查第 k 位

```cpp
(x >> k) & 1
```

把第 `k` 位右移到最低位，再取最低位。

### 把第 k 位置 1

```cpp
x | (1ULL << k)
```

`|` 遇到 `1` 一定变成 `1`，所以第 `k` 位会被打开，其它位保持不变。

### 把第 k 位置 0

```cpp
x & ~(1ULL << k)
```

`~(1ULL << k)` 只有第 `k` 位是 `0`，其它位是 `1`。和它按位与，就只会关闭第 `k` 位。

### 翻转第 k 位

```cpp
x ^ (1ULL << k)
```

`^ 1` 会翻转，`^ 0` 会保持不变，所以只翻转第 `k` 位。

### 取低 k 位全 1

```cpp
(1ULL << k) - 1
```

例如：

```text
(1 << 4) - 1 = 1111(2)
```

注意：当 `k` 等于整数位宽时，`1ULL << k` 会越界。竞赛代码里要保证 `k < 64`，或单独处理边界。

## lowbit

`lowbit(x)` 表示只保留 `x` 二进制中最低位的 `1`：

```cpp
x & -x
```

例如：

```text
x        = 1011000
lowbit  = 0001000
```

这个操作在树状数组中非常重要，也常用于枚举一个集合中的元素。

### 为什么 `x & -x` 能取到最低位 1

用二进制看，`-x` 等于 `~x + 1`。

设 `x` 从低位开始，最低的 `1` 在某个位置：

```text
x    = ?????1000
~x   = ?????0111
~x+1 = ?????1000
```

最低位 `1` 右边全是 `0`，取反后变成全 `1`，再加 `1` 正好进位回这个最低的 `1`。更高位可能变化，但和原数按位与后，只会保留这个最低位 `1`。

## 清除最低位 1

```cpp
x & (x - 1)
```

例如：

```text
110100
110011
------
110000
```

每执行一次，就删掉一个最低位的 `1`。所以可以用它统计 `1` 的个数：

```cpp
int cnt = 0;
while (x) {
    ++cnt;
    x &= x - 1;
}
```

C++ 中也可以直接使用：

```cpp
__builtin_popcountll(x)
```

## 最高位

最高位 `1` 的位置等价于 $\lfloor \log_2 x\rfloor$。

对 `unsigned long long`，可以用：

```cpp
63 - __builtin_clzll(x)
```

其中 `__builtin_clzll(x)` 表示从最高位开始连续 `0` 的数量。注意 `x=0` 时没有最高位，必须单独处理。

只保留最高位：

```cpp
1ULL << highest_bit_pos(x)
```

清除最高位：

```cpp
x ^ keep_highbit(x)
```

## 子集枚举

如果 `mask` 表示一个集合，那么它的子集 `sub` 必须满足：

```cpp
(sub & mask) == sub
```

常用枚举写法：

```cpp
for (int sub = mask; sub; sub = (sub - 1) & mask) {
    // sub 是 mask 的一个非空子集
}
```

如果也要枚举空集，可以在循环外单独处理 `0`，或者使用：

```cpp
for (int sub = mask;; sub = (sub - 1) & mask) {
    // sub
    if (sub == 0) break;
}
```

### 为什么 `(sub - 1) & mask` 能枚举子集

`sub - 1` 会把最低位的 `1` 变成 `0`，并把它右边的位变成 `1`。这相当于找到下一个更小的二进制状态。

但这个状态可能打开了不属于 `mask` 的位，所以再 `& mask`，只保留 `mask` 中允许出现的位。

这样得到的 `sub` 每次都严格变小，因此不会重复，并且会走过所有非空子集。

## 算法步骤

### 操作某一位

1. 构造掩码 `bit = 1ULL << k`。
2. 检查：`x & bit`。
3. 置一：`x | bit`。
4. 清零：`x & ~bit`。
5. 翻转：`x ^ bit`。

### 统计 1 的数量

1. 令 `cnt = 0`。
2. 当 `x != 0`：
   - `cnt++`。
   - `x = x & (x - 1)`，清除最低位的 `1`。
3. `cnt` 就是二进制中 `1` 的数量。

### 枚举子集

1. 令 `sub = mask`。
2. 处理当前 `sub`。
3. 更新 `sub = (sub - 1) & mask`。
4. 当 `sub = 0` 时停止，或单独处理空集。

## 算法证明

### 位操作的正确性

**核心不变量**：掩码 `1ULL << k` 只有第 `k` 位为 `1`。

- `x | mask`：第 `k` 位一定变成 `1`，其它位与 `0` 做或运算，不变。
- `x & ~mask`：第 `k` 位与 `0` 做与运算，变成 `0`，其它位与 `1` 做与运算，不变。
- `x ^ mask`：第 `k` 位与 `1` 做异或，翻转，其它位与 `0` 做异或，不变。

所以这三种操作都只影响第 `k` 位。

### 清除最低位 1 的正确性

设 $x$ 的最低位 `1` 右边有 $t$ 个 `0`：

```text
x     = A 1 00...0
x - 1 = A 0 11...1
```

按位与后：

```text
x & (x - 1) = A 0 00...0
```

更高位 `A` 保持不变，最低位的 `1` 被清除，右侧仍然是 `0`。因此每次操作恰好删掉一个 `1`。

## 复杂度分析

- 单个位操作：时间复杂度 $O(1)$，空间复杂度 $O(1)$。
- `lowbit`、清除最低位、判断 $2$ 的幂：时间复杂度 $O(1)$。
- 统计 `1` 的数量：手写循环为 $O(c)$，其中 $c$ 是 `1` 的个数；内建函数通常视为 $O(1)$。
- 枚举 `mask` 的所有子集：若 `mask` 中有 $c$ 个 `1`，时间复杂度 $O(2^c)$。

## 应用分类详解

位运算的本质是“用一个整数保存很多个二值状态”。看到开关状态、集合选择、二进制拆分、奇偶性、低位贡献时，都应该想到位运算。

### 一、单个数字的二进制性质

**典型模式：** 题目关心奇偶、是否为 $2$ 的幂、最低位 `1`、最高位 `1`、二进制中 `1` 的数量。

**识别信号：** 出现“lowbit”“二进制位数”“最高位”“汉明重量”“power of two”。

**核心建模：** 用位运算直接定位关键位，避免转字符串或循环除法。

| 应用场景 | 经典题目 | 核心思路 |
|---|---|---|
| 判断奇偶 | 基础输入判断 | `x & 1` 判断最低位 |
| 判断 $2$ 的幂 | [LeetCode 231](https://leetcode.cn/problems/power-of-two/) | $2$ 的幂只有一个 `1` |
| 统计 `1` 的个数 | [LeetCode 191](https://leetcode.cn/problems/number-of-1-bits/) | 反复执行 `x &= x - 1` |

### 二、集合状态压缩

**典型模式：** 元素数量不大，每个元素只有选或不选两种状态。

**识别信号：** `n <= 20`、子集、状态压缩、已访问集合、开关集合。

**核心建模：** 第 `i` 位表示第 `i` 个元素是否被选择，整个集合压成一个 `mask`。

| 应用场景 | 经典题目 | 核心思路 |
|---|---|---|
| 子集枚举 | [[problem: luogu,P1157]] | 每个元素对应一位 |
| 状压 DP | [[problem: luogu,P1896]] | 用 `mask` 记录一行或集合状态 |
| SOS DP | 本书 SOS DP 章节 | 在所有二进制集合上做高维前缀和 |

### 三、树状数组与低位块

**典型模式：** 下标要按二进制块跳转。

**识别信号：** 树状数组、前缀和、`i += lowbit(i)`、`i -= lowbit(i)`。

**核心建模：** `lowbit(i)` 是下标 `i` 管理的块长度。更新时跳到下一个包含当前位置的块，查询时不断删掉当前前缀最后一块。

### 四、枚举优化

**典型模式：** 需要枚举所有子集或所有补集，但全集规模较小。

**识别信号：** 对每个集合枚举子集、`3^n` 复杂度、子集卷积、容斥。

**核心建模：** 用 `(sub - 1) & mask` 枚举当前集合的所有子集，只访问合法状态。

## 经典例题

### 1. 判断一个数是否是 2 的幂

若 $x>0$ 且：

```cpp
(x & (x - 1)) == 0
```

则 $x$ 的二进制中只有一个 `1`，所以它是 $2$ 的幂。

### 2. 树状数组中的 lowbit

树状数组每个节点 `i` 维护长度为 `lowbit(i)` 的区间。更新时 `i += lowbit(i)`，查询时 `i -= lowbit(i)`，本质都是在二进制块之间跳转。

### 3. 子集枚举

给定集合 `mask`，枚举它所有非空子集：

```cpp
for (int sub = mask; sub; sub = (sub - 1) & mask) {
    // 处理 sub
}
```

这是状态压缩 DP、SOS DP 和容斥计数中的常用基础操作。
