## 一句话算法

割点判定看 `low[v] >= dfn[u]`：如果子树 `v` 回不到 `u` 的祖先，那么 `u` 就是这棵子树通向外界的关口。

## 问题模型

给定一张无向图。删除某个点以及与它相连的边后，如果图的连通块数量增加，那么这个点叫做割点。

割点反映的是网络中的关键节点：

- 删除它会让某些点互相不可达；
- 它常用于无向图连通性分析、网络脆弱点判断、点双连通分量。

## 核心直觉

DFS 形成一棵搜索树。对一条树边 `u -> v`：

- `dfn[u]` 是 `u` 第一次被访问的时间；
- `low[v]` 是 `v` 的子树通过返祖边能回到的最早祖先。

如果：

$$
low[v] \ge dfn[u]
$$

说明 `v` 的子树最多只能回到 `u`，回不到 `u` 的父亲或更早祖先。删除 `u` 后，`v` 的子树就被切开。

根节点需要特殊判断：DFS 根没有父亲，只有当它有两个及以上 DFS 子树时，它才是割点。

## 算法步骤

1. 对每个未访问的点作为 DFS 根。
2. 进入点 `u` 时，令 `dfn[u] = low[u] = ++timer`。
3. 枚举邻点 `v`：
   - 如果 `v` 未访问，递归 DFS；
   - 回溯后用 `low[v]` 更新 `low[u]`；
   - 若 `u` 不是根且 `low[v] >= dfn[u]`，标记 `u` 为割点。
4. 如果 `v` 已访问且是祖先，用 `dfn[v]` 更新 `low[u]`。
5. DFS 根如果有超过一个 DFS 子树，标记为割点。

## 算法证明

**核心不变量**：`low[u]` 表示 `u` 的子树通过树边和最多一条返祖边能到达的最早时间戳。

1. 初始化时，`low[u] = dfn[u]`，表示至少能到自己。
2. 遇到未访问子节点 `v`，`v` 子树能到达的最早祖先也能被 `u` 使用，所以 `low[u] = min(low[u], low[v])`。
3. 遇到祖先 `v`，边 `(u,v)` 是返祖边，所以 `low[u] = min(low[u], dfn[v])`。
4. 若 `low[v] >= dfn[u]`，说明 `v` 子树不能绕过 `u` 回到更早祖先。删除 `u` 后，`v` 子树与外部断开，因此 `u` 是割点。
5. 根节点没有“外部祖先”，只有当它有两个及以上 DFS 子树时，删除根才会让这些子树互相断开。

因此判定条件正确。

## 复杂度分析

设点数为 $n$，边数为 $m$。

- 时间复杂度：$O(n+m)$。
- 空间复杂度：$O(n+m)$，取决于图存储和递归栈。

## 测试用例

输入图：

```text
5 4
1 2
2 3
3 4
3 5
```

割点是：

```text
2 3
```

删除 `2` 后，点 `1` 与 `3,4,5` 断开；删除 `3` 后，点 `4`、`5` 与前半部分断开。

## 应用分类详解

割点用于识别无向图中的关键点。

### 一、网络脆弱点

**典型模式：** 删除某个节点后，连通性变差。

**识别信号：** 题面出现“关键城市”“通信站”“删除点后不连通”。

**核心建模：** 把对象看作点，连接关系看作无向边，求割点。

| 应用场景 | 经典题目 | 核心思路 |
|---|---|---|
| 割点模板 | [[problem: luogu,P3388]] | Tarjan 求所有割点 |
| 通信网络关键点 | 网络可靠性题 | 删除点后连通块增加 |

### 二、点双连通分量的入口

**典型模式：** 求每个点双，或者分析哪些点属于多个点双。

**识别信号：** 出现“点双连通”“任意两点有两条点不重复路径”。

**核心建模：** 割点是点双之间的连接处。
