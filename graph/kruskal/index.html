Kruskal

性质: MST增量更新

在处理动态图或增量更新的最小生成树问题时，有一个非常深刻且有用的性质。这个性质是许多优化算法（例如题目 csp-s 2025 T2 道路修复 中可能用到的方法）的理论基础。

性质描述

假设我们有一个边集 eme_mem​ 和另一个边集 ese_ses​。我们想要计算 eme_mem​ 和 ese_ses​ 并集的最小生成树。该性质指出，我们可以先计算 eme_mem​ 的最小生成树，然后用这个结果和 ese_ses​ 一起计算最终的最小生成树。

用数学公式表达为：

Cost(MST(em∪eS))=Cost(MST(MST(em)∪eS)) \text{Cost}(\text{MST}(\text{e}_m \cup \text{e}_S)) = \text{Cost}(\text{MST}(\text{MST}(\text{e}_m) \cup \text{e}_S)) Cost(MST(em​∪eS​))=Cost(MST(MST(em​)∪eS​))

直观理解

这个性质的直观理解是：

任何不在 eme_mem​ 的最小生成树里的边（我们称之为“无用边”），都因为它在 eme_mem​ 内部的某个环路中是“最差”（最重）的边。

后来加入 ese_ses​ 可能会提供 新 的环路，但这并不会改变那条“无用边”在它 原来 的环路中是“最差”的事实。

既然 eme_mem​ 内部已经有了一条更优的路径（MST(em)MST(e_m)MST(em​) 中的路径）来连接那条“无用边”的两个端点，那么在求全局MST时，算法就永远不会选择那条“无用边”，因为它总能找到更优的替代方案。

证明

为了方便，我们定义几个术语：

  • Em=MST(em)E_m = \text{MST}(\text{e}_m)Em​=MST(em​)：只使用 eme_mem​ 算出的MST的边的集合。
  • Em′=em∖EmE_m' = \text{e}_m \setminus E_mEm′​=em​∖Em​：所有 eme_mem​ 中没有被 EmE_mEm​ 选中的边（“无用边”）。
  • G1=em∪eS=Em∪Em′∪eSG_1 = \text{e}_m \cup \text{e}_S = E_m \cup E_m' \cup \text{e}_SG1​=em​∪eS​=Em​∪Em′​∪eS​： 包含所有边的完整图。
  • G2=Em∪eSG_2 = E_m \cup \text{e}_SG2​=Em​∪eS​： 只包含 EmE_mEm​ 和新边集的“优化图”。
  • T1=MST(G1)T_1 = \text{MST}(G_1)T1​=MST(G1​)：完整图的MST（这是我们真正想求的）。
  • T2=MST(G2)T_2 = \text{MST}(G_2)T2​=MST(G2​)：“优化图”的MST。

我们要证明的是：Cost(T1)=Cost(T2)\text{Cost}(T_1) = \text{Cost}(T_2)Cost(T1​)=Cost(T2​)。

这个证明分为两个部分：

1. 证明 Cost(T1)⩽Cost(T2)\text{Cost}(T_1) \leqslant \text{Cost}(T_2)Cost(T1​)⩽Cost(T2​)

这部分很简单。

  • G2G_2G2​ 的边集（Em∪eSE_m \cup \text{e}_SEm​∪eS​）是 G1G_1G1​ 边集（Em∪Em′∪eSE_m \cup E_m' \cup \text{e}_SEm​∪Em′​∪eS​）的一个子集。
  • T2T_2T2​ 是 G2G_2G2​ 的一个最小生成树。
  • T2T_2T2​ 本身也是 G1G_1G1​ 的一个（不一定是最小的）生成树，因为它连接了所有节点。
  • T1T_1T1​ 是 G1G_1G1​ 的最小生成树。根据定义，T1T_1T1​ 的成本必须小于或等于 G1G_1G1​ 中任何其他生成树的成本。
  • 因此，Cost(T1)⩽Cost(T2)\text{Cost}(T_1) \leqslant \text{Cost}(T_2)Cost(T1​)⩽Cost(T2​)。

一句话: 集合大的小

另一种:

  1. 因为集合G2⊆G1G_2 \subseteq G_1G2​⊆G1​,考虑幂集,得出P(G2)⊆P(G1)P(G_2) \subseteq P(G_1)P(G2​)⊆P(G1​)
  2. Cost(T1)Cost(T_1)Cost(T1​)本质是符合条件的幂集的最小值
  3. 任何在P(G2)P(G_2)P(G2​)的最小值都会出现在P(G1)P(G_1)P(G1​)中
  4. 显然，Cost(T1)⩽Cost(T2)\text{Cost}(T_1) \leqslant \text{Cost}(T_2)Cost(T1​)⩽Cost(T2​)。

2. 证明 Cost(T1)⩾Cost(T2)\text{Cost}(T_1) \geqslant \text{Cost}(T_2)Cost(T1​)⩾Cost(T2​)

这是证明的关键。我们将使用一个“交换论证”（Exchange Argument）。

我们从 T1T_1T1​（完整图的MST）出发，一步一步地把它变成一个只使用 G2G_2G2​ 中边的生成树，并且成本不会增加。

分情况讨论:

情况1: 只由G2G_2G2​中的边组成

如果 T1T_1T1​ 已经只包含 G2G_2G2​ 中的边（即 T1T_1T1​ 中没有任何来自 Em′E_m'Em′​ 的边），那么 T1T_1T1​ 本身就是 G2G_2G2​ 的一个生成树。因为 T2T_2T2​ 是 G2G_2G2​ 的 最小 生成树，所以 Cost(T1)⩾Cost(T2)\text{Cost}(T_1) \geqslant \text{Cost}(T_2)Cost(T1​)⩾Cost(T2​)。结合第1部分，我们得到 Cost(T1)=Cost(T2)\text{Cost}(T_1) = \text{Cost}(T_2)Cost(T1​)=Cost(T2​)。

情况2: 先断开,然后重新连

  1. 如果 T1T_1T1​ 中包含了一条（或多条）来自 Em′E_m'Em′​ 的边，我们任选一条这样的边 e=(u,v)e = (u, v)e=(u,v)。

    • eee 在 eme_mem​ 中，但 e∉Eme \notin E_me∈/Em​(e在原图中,但是不在原图MST中)。
    • 根据MST的环路性质：将 eee 添加到 EmE_mEm​ 中，必然会形成一个环路 CCC。
    • 在这个环路 CCC 中， eee 一定是权重最大（或之一）的边。环路 CCC 上的所有其它边 eie_iei​ 都满足 Cost(ei)⩽Cost(e)\text{Cost}(e_i) \leqslant \text{Cost}(e)Cost(ei​)⩽Cost(e)，并且所有这些 eie_iei​ 都在 EmE_mEm​ 中。

  2. 现在，我们回到 T1T_1T1​。T1T_1T1​ 是一个树，它包含了 eee。

    • 如果我们从 T1T_1T1​ 中移除 eee， T1T_1T1​ 会断裂成两个连通分量 AAA 和 BBB（u∈A,v∈Bu \in A, v \in Bu∈A,v∈B）。
    • 但是，我们在上面知道 eee 所在的环路 CCC 中，除了 eee 之外，还有一条 只由 EmE_mEm​ 中的边构成的 路径 P=C∖{e}P = C \setminus \{e\}P=C∖{e} 也能连接 uuu 和 vvv。
    • 由于 PPP 上的所有边都属于 EmE_mEm​，而 T1T_1T1​ 也是一个连接所有节点的生成树， T1T_1T1​ 不可能包含了 PPP 上的所有边（否则 T1T_1T1​ 中会有一个环）。
    • 因此，PPP 上必定存在至少一条边 eke_kek​，它的两个端点分别位于 AAA 和 BBB 中。

  3. 让我们来构造一个新的生成树 T1′=(T1∖{e})∪{ek}T_1' = (T_1 \setminus \{e\}) \cup \{e_k\}T1′​=(T1​∖{e})∪{ek​}。

    • T1′T_1'T1′​ 也是一个生成树（我们断开了 eee，又用 eke_kek​ 连上了）。
    • 我们来比较成本：Cost(T1′)=Cost(T1)−Cost(e)+Cost(ek)\text{Cost}(T_1') = \text{Cost}(T_1) - \text{Cost}(e) + \text{Cost}(e_k)Cost(T1′​)=Cost(T1​)−Cost(e)+Cost(ek​)。
    • 因为 eke_kek​ 在环路 CCC 上，我们从第2步知道 Cost(ek)≤Cost(e)\text{Cost}(e_k) \le \text{Cost}(e)Cost(ek​)≤Cost(e)。
    • 因此，Cost(T1′)≤Cost(T1)\text{Cost}(T_1') \le \text{Cost}(T_1)Cost(T1′​)≤Cost(T1​)。

  4. 我们成功地将 eee（一条 Em′E_m'Em′​ 中的边）替换为了 eke_kek​（一条 EmE_mEm​ 中的边），并且总成本没有增加（反而可能降低了）。

    • 由于 T1T_1T1​ 本来就是 最小生成树，它的成本不可能再降低了，所以必定有 Cost(T1′)=Cost(T1)\text{Cost}(T_1') = \text{Cost}(T_1)Cost(T1′​)=Cost(T1​)。
    • 这意味着 T1′T_1'T1′​ 也是 G1G_1G1​ 的一个最小生成树。

  5. 我们可以对 T1T_1T1​ 中所有来自 Em′E_m'Em′​ 的边重复这个“交换”过程。最终，我们会得到一个新的最小生成树 T1∗T_1^*T1∗​，它具有 Cost(T1∗)=Cost(T1)\text{Cost}(T_1^*) = \text{Cost}(T_1)Cost(T1∗​)=Cost(T1​)，并且 T1∗T_1^*T1∗​ 只包含来自 EmE_mEm​ 和 ese_ses​ 的边。

    • 换句话说，T1∗T_1^*T1∗​ 是一个完全由 G2G_2G2​ 中的边构成的生成树。(这就变成了情况1)

  6. 因为 T1∗T_1^*T1∗​ 是 G2G_2G2​ 的一个生成树，而 T2T_2T2​ 是 G2G_2G2​ 的最小生成树，所以 Cost(T2)⩽Cost(T1∗)\text{Cost}(T_2) \leqslant \text{Cost}(T_1^*)Cost(T2​)⩽Cost(T1∗​)。

  7. 结合 Cost(T1∗)=Cost(T1)\text{Cost}(T_1^*) = \text{Cost}(T_1)Cost(T1∗​)=Cost(T1​)，我们得到 Cost(T2)≤Cost(T1)\text{Cost}(T_2) \le \text{Cost}(T_1)Cost(T2​)≤Cost(T1​)。

结论

  • 在第1部分，我们证明了 Cost(T1)⩽Cost(T2)\text{Cost}(T_1) \leqslant \text{Cost}(T_2)Cost(T1​)⩽Cost(T2​)。
  • 在第2部分，我们证明了 Cost(T1)⩾Cost(T2)\text{Cost}(T_1) \geqslant \text{Cost}(T_2)Cost(T1​)⩾Cost(T2​)。

唯一的可能是 Cost(T1)=Cost(T2)\text{Cost}(T_1) = \text{Cost}(T_2)Cost(T1​)=Cost(T2​)。

证明完毕。 这意味着我们可以安全地丢弃所有 Em′E_m'Em′​ 中的边，只在 Em∪eSE_m \cup \text{e}_SEm​∪eS​ 上运行MST算法，这极大地降低了边的数量，从而优化了算法。
