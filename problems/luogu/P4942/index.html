1 核心观察 (Key Observation)

题目要求我们将 lll 到 rrr 之间的所有整数拼接成一个大数 NNN，并求 N(mod9)N \pmod 9N(mod9)。 例如：l=2,r=5  ⟹  N=2345l=2, r=5 \implies N=2345l=2,r=5⟹N=2345。

我们利用 模 9 的整除特征：

一个数模 9 的余数，等于其各位数字之和模 9 的余数。

x≡S(x)(mod9)x \equiv S(x) \pmod 9x≡S(x)(mod9)

证明:

这是解题最关键的一步。

  • 知识点： 一个十进制数 nnn 与其各位数字之和在模 999 的意义下是同余的。

  • 数学表达： 设 S(n)S(n)S(n) 为 nnn 的各位数字之和，则：

    n≡S(n)(mod9)n \equiv S(n) \pmod 9n≡S(n)(mod9)

  • 原理： 因为 10≡1(mod9)10 \equiv 1 \pmod 910≡1(mod9)，所以 10k≡1k≡1(mod9)10^k \equiv 1^k \equiv 1 \pmod 910k≡1k≡1(mod9)(同余相乘性质)。 对于任意数字，例如 357=3×100+5×10+7357 = 3 \times 100 + 5 \times 10 + 7357=3×100+5×10+7，在模 9 下：

    3×100+5×10+7≡3×1+5×1+7≡3+5+7(mod9)3 \times 100 + 5 \times 10 + 7 \equiv 3 \times 1 + 5 \times 1 + 7 \equiv 3+5+7 \pmod 93×100+5×10+7≡3×1+5×1+7≡3+5+7(mod9)

这一性质将“拼接问题”转化为了“求和问题”。

  • 转化： 所有数位之和，其实就是把 l,l+1,…,rl, l+1, \dots, rl,l+1,…,r 每个数字拆开相加。 由第 1 点可知，每个数字 xxx 的数位和 ≡x(mod9)\equiv x \pmod 9≡x(mod9)。 结论： 拼接起来的大数 NNN 模 9 的余数，等于数列 lll 到 rrr 的数值之和模 9 的余数。
    Answer=(∑i=lri) mod 9\text{Answer} = \left( \sum_{i=l}^{r} i \right) \bmod 9Answer=(i=l∑r​i)mod9

题目核心
拼接数 NNN 模 9 的余数 === 等差数列 [l,r][l, r][l,r] 的和模 9 的余数。
把拼接转换成求和，从而简化问题。

2 数学推导 (Derivation)

我们需要计算等差数列和：

Sum=(l+r)(r−l+1)2\text{Sum} = \frac{(l+r)(r-l+1)}{2}Sum=2(l+r)(r−l+1)​
最终答案为 Sum mod 9\text{Sum} \bmod 9Summod9。

由于 l,r≤1012l, r \le 10^{12}l,r≤1012，直接计算乘积会爆 long long，且我们涉及除法运算，不能直接对分子取模。这里有两种处理方法。

方法一：乘法逆元 (Modular Inverse) —— 推荐

在模运算中，除以一个数等价于乘以这个数的逆元。 我们需要计算 /2(mod9)/ 2 \pmod 9/2(mod9)。因为 gcd⁡(2,9)=1\gcd(2, 9) = 1gcd(2,9)=1，逆元存在。 寻找一个 xxx 使得 2x≡1(mod9)2x \equiv 1 \pmod 92x≡1(mod9)。显而易见 2×5=10≡1(mod9)2 \times 5 = 10 \equiv 1 \pmod 92×5=10≡1(mod9)。

所以，除以 2 等价于乘以 5。 公式转化为：

Ans=[(l+r)×(r−l+1)×5] mod 9\text{Ans} = \left[ (l+r) \times (r-l+1) \times 5 \right] \bmod 9Ans=[(l+r)×(r−l+1)×5]mod9

这种方法全程取模，完全不需要大整数类型，也不用担心溢出。

方法二：扩展模数 (Expanding Modulus)

如果我们不知道逆元，也可以利用性质：

若 A/2=B(mod9)A/2 = B \pmod 9A/2=B(mod9)，则 A=2B(mod18)A = 2B \pmod{18}A=2B(mod18)。

原理是：我们需要求 BBB，而 B<9B < 9B<9，这保证了 2B<182B < 182B<18。 所以 2B(mod18)2B \pmod{18}2B(mod18) 的结果就是 2B2B2B 本身（不会发生截断）。我们只需要算出分子对 181818 取模的结果，然后直接除以 2 即可还原出 BBB。

公式转化为：

Ans=[(l+r)(r−l+1) mod 18]/2\text{Ans} = \left[ (l+r)(r-l+1) \bmod{18} \right] / 2Ans=[(l+r)(r−l+1)mod18]/2

解法2: 更神奇的性质

直接暴力,找规律.

观察到，

任意连续 9 个数字的和，模 9 的余数为 0。 例如：1+2+3+4+5+6+7+8+9=45≡0(mod9)1+2+3+4+5+6+7+8+9=45 \equiv 0 \pmod 91+2+3+4+5+6+7+8+9=45≡0(mod9)。

证明:

(n+1)+(n+2)+⋯+(n+9)=9n+(1+2+⋯+9)=9n+45≡0(mod9) (n+1) + (n+2) + \dots + (n+9) = 9n + (1+2+\dots+9) = 9n + 45 \equiv 0 \pmod 9 (n+1)+(n+2)+⋯+(n+9)=9n+(1+2+⋯+9)=9n+45≡0(mod9)

因此，我们可以将区间 [l,r][l, r][l,r] 分成若干个长度为 9 的完整区间，以及剩余的部分。 完整区间的和模 9 为 0，只需要计算剩余部分的和即可。

        #include <iostream>

using namespace std;

int main() {
    // 优化输入输出
    ios::sync_with_stdio(false);
    cin.tie(0);

    int Q;
    cin >> Q;
    while (Q--) {
        long long l, r;
        cin >> l >> r;

        // 1. 算出总长度
        long long len = r - l + 1;
        
        // 2. 算出除以9之后的余数（也就是需要计算的数的个数）
        //    如果能被9整除，remain就是0，循环都不用进，直接输出0
        long long remain = len % 9;
        
        long long ans = 0;
        
        // 3. 只计算这剩下的几个数即可
        //    循环最多跑 8 次，飞快！
        for (int i = 0; i < remain; i++) {
            ans = (ans + (l + i)) % 9;
        }
        
        cout << ans << "\n";
    }
    return 0;
}

解法3: 不用求逆元

这是一份针对你这段代码的简洁版题解，适合放在博客或题解区。它强调了代码中“奇偶性判断”的巧妙之处。

题目要求计算 lll 到 rrr 所有数字拼接后的模 9 余数。根据模 9 的同余性质，这等价于求 数列 [l,r][l, r][l,r] 的和模 9。

我们使用等差数列求和公式：

Sum=(l+r)×(r−l+1)2\text{Sum} = \frac{(l+r) \times (r-l+1)}{2}Sum=2(l+r)×(r−l+1)​

这段代码并没有直接套用公式计算（因为直接相乘会爆 long long），而是采用了一个非常聪明的**“先除后乘”**策略。

设 A=l+rA = l + rA=l+r（首尾和），B=r−l+1B = r - l + 1B=r−l+1（项数）。 公式即为：A×B2\frac{A \times B}{2}2A×B​。

关键点：奇偶性判断

因为数列的和一定是整数，所以分子 A×BA \times BA×B 一定是偶数。这意味着 AAA 和 BBB 中至少有一个是偶数。
