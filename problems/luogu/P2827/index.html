        #include <iostream>
#include <vector>
#include <algorithm> // 用于 max_element
#include <cmath>     // 用于 floor

using namespace std;

int main() {
    // 优化一下输入输出，虽然暴力算法本身慢，但输入习惯要好
    ios::sync_with_stdio(false);
    cin.tie(NULL);

    int n, m, q, u, v, t;
    cin >> n >> m >> q >> u >> v >> t;

    // 使用 vector 存储当前所有的蚯蚓长度
    vector<int> worms;
    for (int i = 0; i < n; i++) {
        int temp;
        cin >> temp;
        worms.push_back(temp);
    }

    // --- 模拟开始，循环 m 秒 ---
    for (int i = 1; i <= m; i++) {
        
        // [步骤 1: 找出最大的蚯蚓]
        // max_element 会遍历整个 vector 找到最大值的迭代器
        auto max_it = max_element(worms.begin(), worms.end());
        int max_val = *max_it; // 取出最大值的数值

        // [步骤 2: 将其从队伍中取出/移除]
        // 这一步非常慢，因为 vector 删除元素需要移动后面的所有数据
        worms.erase(max_it);

        // 按题目要求，每隔 t 秒输出一次被切断蚯蚓的长度
        if (i % t == 0) {
            cout << max_val << " ";
        }

        // [步骤 3: 这里的代码完成了 "其余蚯蚓增加 q"]
        // 注意：此时最大的那只已经被 erase 移除了，新的两只还没放进去。
        // 所以现在 vector 里剩下的，就是所有 "其余的蚯蚓"。
        for (int &w : worms) {
            w += q; 
        }

        // [步骤 4: 切割产生两只新蚯蚓]
        // 计算长度，注意整数运算
        // 题目公式：长度1 = floor(x * u / v)
        long long len1_long = (long long)max_val * u / v; 
        int len1 = (int)len1_long;
        int len2 = max_val - len1;

        // [步骤 5: 将新蚯蚓放回队伍]
        // 题目说：新产生的两只蚯蚓这一秒不增加长度，所以它们是在上面的 for 循环之后加入的
        worms.push_back(len1);
        worms.push_back(len2);
    }

    cout << endl; // 第一行输出结束

    // [步骤 6: 输出最后的结果]
    // 题目要求输出第 m 秒结束后，所有蚯蚓长度（从大到小）
    // 暴力解法直接排序即可
    sort(worms.begin(), worms.end(), greater<int>());

    for (int i = 0; i < worms.size(); i++) {
        if ((i + 1) % t == 0) {
            cout << worms[i] << " ";
        }
    }
    cout << endl;

    return 0;
}

题目解析

这是一道非常经典的NOIP提高组题目: [NOIP 2016 提高组] 蚯蚓，它考察了对数据结构的优化以及对题目性质（单调性）的挖掘。

1. 朴素思路（只拿部分分）

题目要求每次取出最大的蚯蚓进行切割，并将其他蚯蚓长度增加 qqq。

最直观的想法是使用 优先队列（大根堆）。

  • 每次从堆顶取出最大值 xxx。
  • 切割成 ⌊px⌋\lfloor px \rfloor⌊px⌋ 和 x−⌊px⌋x - \lfloor px \rfloorx−⌊px⌋。
  • 关于“其余蚯蚓增加 qqq”的处理：
    • 如果每次都遍历堆中所有元素加 qqq，时间复杂度爆炸。
    • 我们可以维护一个全局变量 delta，表示累加增长量。
    • 放入堆时，将数值减去当前的 delta；取出时，将数值加上 delta 还原为真实长度。
  • 复杂度分析：
    • 操作次数 mmm 最大为 7×1067 \times 10^67×106。
    • 堆的插入删除是 O(log⁡N)O(\log N)O(logN)。总复杂度 O(mlog⁡(n+m))O(m \log (n+m))O(mlog(n+m))。
    • 算一下：7×106×log⁡2(7×106)≈7×106×23≈1.6×1087 \times 10^6 \times \log_2(7 \times 10^6) \approx 7 \times 10^6 \times 23 \approx 1.6 \times 10^87×106×log2​(7×106)≈7×106×23≈1.6×108。
    • 对于 1秒的时限来说，这个计算量非常危险，可能会 TLE（超时），只能拿 60~85 分左右。

2. 正解思路（利用单调性）

为了拿到 100 分，我们需要 O(1)O(1)O(1) 取出最大值，而不是 O(log⁡N)O(\log N)O(logN)。这意味着我们需要利用题目中的隐含规律。

关键性质：

如果在 ttt 时刻切断了一只长为 xxx 的蚯蚓，生成了 x1,x2x_1, x_2x1​,x2​。

如果在 t+1t+1t+1 时刻切断了一只长为 yyy 的蚯蚓（且 x≥yx \ge yx≥y），生成了 y1,y2y_1, y_2y1​,y2​。

那么一定满足：先切出来的部分 ≥\ge≥ 后切出来的部分。

（即 x1+q≥y1x_1 + q \ge y_1x1​+q≥y1​ 且 x2+q≥y2x_2 + q \ge y_2x2​+q≥y2​，哪怕考虑生长因素，这个大小关系依然成立）。

推导结论：

因为切分出来的两段长度随时间推移是单调递减的（相对于产生的时刻），我们可以不用堆，而是用 3个普通的队列 来维护：

  1. Q1Q_1Q1​：存储初始给定的 nnn 条蚯蚓（先从大到小排好序）。
  2. Q2Q_2Q2​：存储切分出来的左半段长度（⌊px⌋\lfloor px \rfloor⌊px⌋）。
  3. Q3Q_3Q3​：存储切分出来的右半段长度（x−⌊px⌋x - \lfloor px \rfloorx−⌊px⌋）。

由于单调性，Q2Q_2Q2​ 和 Q3Q_3Q3​ 里的元素天然就是从大到小排列的，Q1Q_1Q1​ 我们也预先排好了。

每次最大的蚯蚓，一定就在这三个队列的队头产生。

3. 算法流程

  1. 将初始蚯蚓排序，放入 Q1Q_1Q1​。
  2. 循环 mmm 次（时间 t=1…mt=1 \dots mt=1…m）：
    • 比较 Q1,Q2,Q3Q_1, Q_2, Q_3Q1​,Q2​,Q3​ 三个队列的队头元素，取出最大的那个（记为 max_lenmax\_lenmax_len）。
    • 还原真实长度：real_len=max_len+(t−1)×qreal\_len = max\_len + (t-1) \times qreal_len=max_len+(t−1)×q。
    • 按要求输出。
    • 计算切分长度 l=⌊real_len×u/v⌋l = \lfloor real\_len \times u / v \rfloorl=⌊real_len×u/v⌋, r=real_len−lr = real\_len - lr=real_len−l。
    • 将 lll 和 rrr 减去 当前的总增长量 t×qt \times qt×q，分别放入 Q2Q_2Q2​ 和 Q3Q_3Q3​ 的队尾（为了统一处理增长，新来的要“亏”掉之前的增长）。
  3. 循环结束后，将三个队列剩余的元素合并，排序（或者直接利用归并排序的思想输出），得到最终结果。

易错点解析

  1. 数据类型：题目中 aia_iai​ 最大 10810^8108，经过 7×1067 \times 10^67×106 秒，每秒长 200，最大长度可能达到 108+1.4×109≈1.5×10910^8 + 1.4 \times 10^9 \approx 1.5 \times 10^9108+1.4×109≈1.5×109。虽然 int 范围是 2.1×1092.1 \times 10^92.1×109 勉强够用，但在计算 real_len * u 时，如果 uuu 很大，109×10910^9 \times 10^9109×109 必定溢出 int，所以中间计算必须强制转为 long long。
  2. 相对长度技巧：这是处理“全员加减”的经典技巧。记录一个偏移量 offsetoffsetoffset，入队时存 val - offset，出队时用 val + offset。
  3. 空行输出：题目要求“即使某一行没有任何数需要输出，你也应输出一个空行”。cout << "\n" 必须要有，否则可能格式错误。
  4. 手动队列：由于 mmm 很大，STL 的 queue 涉及大量的动态内存分配，可能导致常数过大超时或内存碎片。使用数组模拟队列 q[head], q[tail] 是最稳妥的。

这张图展示了三个队列如何像三条传送带一样，每次只需看三个“出口”处的最大值，即可保证全局最大，这就是 O(m)O(m)O(m) 的精髓。

证明

一、 “全局加 q”的数学证明：利用加法的逆元性质

核心思想：我们利用了相对论的思想。如果每个人都长高了，等于没人长高而坐标系降低了。

1. 问题定义

我们需要维护一个集合 SSS，支持两种操作：

  1. Extract: 取出最大值 xxx。
  2. Global Add: 集合中所有剩下的元素 yyy 变为 y+qy + qy+q。

2. 朴素操作 vs. 优化操作

假设第 iii 次操作发生在时刻 tit_iti​（从 0 开始计数）。

对于一个在时刻 tint_{in}tin​ 加入集合的初始长度为 valvalval 的数值，在时刻 tnowt_{now}tnow​ 取出时，它实际经历的增长次数是 tnow−tint_{now} - t_{in}tnow​−tin​。

也就是：

RealValue=val+(tnow−tin)×qRealValue = val + (t_{now} - t_{in}) \times qRealValue=val+(tnow​−tin​)×q

3. 变换公式（利用逆元）

我们展开上面的公式：

RealValue=val−tin×q+tnow×qRealValue = val - t_{in} \times q + t_{now} \times qRealValue=val−tin​×q+tnow​×q

这里我们发现了一个变量分离的技巧：

  • val−tin×qval - t_{in} \times qval−tin​×q：这一部分只和进入集合的时间有关，进入后是一个常数。
  • tnow×qt_{now} \times qtnow​×q：这一部分只和当前的时刻有关，对集合里所有元素都是一样的。

结论：

我们不需要更新集合里的每一个数。

  • 入队时：存入 val′=RealLength−CurrentGlobalDeltaval' = RealLength - CurrentGlobalDeltaval′=RealLength−CurrentGlobalDelta。
  • 出队时：读出 RealLength=val′+CurrentGlobalDeltaRealLength = val' + CurrentGlobalDeltaRealLength=val′+CurrentGlobalDelta。

这在数学群论中，利用了加法群的逆元性质：为了抵消未来的增长，我们先减去当前的累积量（预支亏损），未来再加上累积量时，差值正好就是这段时间的增长。

二、 单调性是如何被“普通人”发现的？（观察与思维跳跃）

你问到普通人如何从朴素思路跳跃到“单调性”？通常这不是灵光一闪，而是通过小数据模拟和极端情况分析得到的。

1. 直觉来源：通胀模型

想象“长度”是钱，“增长 qqq”是政府发的福利，“切割”是收税。

  • 早切（早分家）：你手里的大钱马上变成了两笔小钱。但是，这两笔小钱从下一秒开始，每一笔都能领到完整的福利 qqq。
  • 晚切（晚分家）：你死守着大钱，虽然大钱本身领了一份福利 qqq，但当你晚一点被“切”的时候，这一份福利 qqq 也被切成了两半（比如乘上 ppp）。

直观感受：早切的人，虽然本金变小了，但享受福利的“份数”变多了（一份变两份，每份都拿 qqq）。晚切的人，福利也被稀释了。所以，早切产生的部分 ≥\ge≥ 晚切产生的部分。

2. 观察步骤（模拟）

假设 p=0.5p=0.5p=0.5（对半切），q=10q=10q=10。

  • A蚯蚓（长度 100）：现在切 →\rightarrow→ 变成 50, 50。
    • 过 1 秒后，它们变成：60, 60。
  • B蚯蚓（长度 98）：现在不切，过 1 秒切。
    • 过 1 秒后，B 先长到 98+10 = 108。
    • 然后切 →\rightarrow→ 变成 54, 54。

发现规律：

虽然 A (100) 和 B (98) 只差 2，但 A 早切了一秒，结果 A 的后代 (60) 比 B 的后代 (54) 大了很多。

这就验证了猜想：先切出来的，一定比后切出来的长。

直觉模型：分家产模型

早分家 vs 晚分家

  • 早分家（切 xxx）：

    大家立刻独立。你分到的小家产，下一秒能完整领到一份新的补助 qqq。

    收益 = 完整补助

  • 晚分家（切 yyy）：

    大家先不分，作为大家庭领了一份补助 qqq。

    下一秒分家时，这一份补助也被劈成了两半（乘 ppp）。你只能分到残缺的补助 pqpqpq。

    收益 = 打折补助

结论：早分家的人，领到的补助多。

极简算式推导流

设定：当前最大值 xxx，次大值 yyy。

已知：x≥yx \ge yx≥y，比例 0<p<10 < p < 10<p<1，增长 q>0q > 0q>0。

路径 A：早切 xxx

  1. 切：变 pxpxpx

  2. 长：变 px+qpx + qpx+q

    结果 A =px+q= px + q=px+q

路径 B：晚切 yyy

  1. 长：变 y+qy + qy+q

  2. 切：变 p(y+q)p(y + q)p(y+q)

    结果 B =py+pq= py + pq=py+pq

=(px+q)−(py+pq)= (px + q) - (py + pq)=(px+q)−(py+pq)

=p(x−y)⏟老本优势≥0+q(1−p)⏟补助优势≥0= \underbrace{p(x - y)}_{\text{老本优势} \ge 0} + \underbrace{q(1 - p)}_{\text{补助优势} \ge 0}=老本优势≥0p(x−y)​​+补助优势≥0q(1−p)​​

三、 单调性的严格数学证明

这里我们需要证明：对于从队列中先后取出的两个最大值 xxx 和 yyy（假设 xxx 先被取出，满足 x≥yx \ge yx≥y），xxx 分裂并生长后的长度，一定大于等于 yyy 生长并分裂后的长度。

设定变量

  • ttt 时刻，取出最大值 xxx。
  • t+1t+1t+1 时刻，取出剩下的最大值 yyy（注意：这个 yyy 在 ttt 时刻就在队伍里，或者它也是刚被切出来的，但由于我们前提是“有序队列”，我们总能保证 x≥yx \ge yx≥y。如果是 yyy 在 ttt 时刻还没生成，那它自然更小，不用证）。
  • 切割比例参数为 ppp (0<p<10 < p < 10<p<1)。
  • 每秒增长量为 qqq。

证明目标

我们需要证明切出来的较长那一段满足单调性（较短那一段同理）。

  1. xxx 的轨迹：

    在 ttt 时刻被切，较长一段长度为 ⌊px⌋\lfloor px \rfloor⌊px⌋。

    到了 t+1t+1t+1 时刻，它长了 qqq，长度变为：

    Lx=⌊px⌋+qL_x = \lfloor px \rfloor + qLx​=⌊px⌋+q

  2. yyy 的轨迹：

    在 ttt 时刻没被切，先长了 qqq，长度变成 y+qy+qy+q。

    在 t+1t+1t+1 时刻被切，较长一段长度变为：

    Ly=⌊p(y+q)⌋L_y = \lfloor p(y+q) \rfloorLy​=⌊p(y+q)⌋

求证： Lx≥LyL_x \ge L_yLx​≥Ly​，即 ⌊px⌋+q≥⌊p(y+q)⌋\lfloor px \rfloor + q \ge \lfloor p(y+q) \rfloor⌊px⌋+q≥⌊p(y+q)⌋

证明过程

利用取整函数的性质：x−1<⌊x⌋≤xx - 1 < \lfloor x \rfloor \le xx−1<⌊x⌋≤x。

我们比较不等式左右两边：

左边 =⌊px⌋+q= \lfloor px \rfloor + q=⌊px⌋+q

右边 =⌊py+pq⌋= \lfloor py + pq \rfloor=⌊py+pq⌋

步骤 1：去除取整符号进行放缩（最坏情况分析）

为了证明 A≥BA \ge BA≥B，我们只需证明 Amin≥BmaxA_{min} \ge B_{max}Amin​≥Bmax​ 吗？不完全是，这里用代数变形更严谨。

考虑 ⌊px⌋+q\lfloor px \rfloor + q⌊px⌋+q 与 ⌊py+pq⌋\lfloor py + pq \rfloor⌊py+pq⌋ 的差：

由于 x≥yx \ge yx≥y，且 p≥0p \ge 0p≥0，所以 px≥pypx \ge pypx≥py。

我们展开右边：

⌊p(y+q)⌋=⌊py+pq⌋\lfloor p(y+q) \rfloor = \lfloor py + pq \rfloor⌊p(y+q)⌋=⌊py+pq⌋

因为 ⌊a+b⌋≤⌊a⌋+⌈b⌉\lfloor a+b \rfloor \le \lfloor a \rfloor + \lceil b \rceil⌊a+b⌋≤⌊a⌋+⌈b⌉ 或者更简单地，直接比较核心值。

让我们做一下差值分析：

Left−Right=(⌊px⌋+q)−⌊py+pq⌋\text{Left} - \text{Right} = (\lfloor px \rfloor + q) - \lfloor py + pq \rfloorLeft−Right=(⌊px⌋+q)−⌊py+pq⌋

利用性质 ⌊A⌋+B=⌊A+B⌋\lfloor A \rfloor + B = \lfloor A+B \rfloor⌊A⌋+B=⌊A+B⌋ (当 B 是整数时)。

Left=⌊px+q⌋\text{Left} = \lfloor px + q \rfloorLeft=⌊px+q⌋

现在比较 ⌊px+q⌋\lfloor px + q \rfloor⌊px+q⌋ 和 ⌊py+pq⌋\lfloor py + pq \rfloor⌊py+pq⌋。

我们要证：

px+q≥py+pqpx + q \ge py + pqpx+q≥py+pq
p(x−y)+q(1−p)≥0p(x-y) + q(1-p) \ge 0p(x−y)+q(1−p)≥0

分析这一步：

  1. p(x−y)p(x-y)p(x−y)：因为 x≥yx \ge yx≥y 且 p>0p > 0p>0，所以这一项 ≥0\ge 0≥0。
  2. q(1−p)q(1-p)q(1−p)：因为 q≥0q \ge 0q≥0 且 p<1p < 1p<1（切割比例通常小于1），所以这一项 ≥0\ge 0≥0。

结论：

px+q≥py+pqpx + q \ge py + pqpx+q≥py+pq

这意味着 xxx 切割并生长后的“核心值”确实大于 yyy 生长并切割后的“核心值”。

关于取整符号的微小扰动：

虽然核心值满足 ≥\ge≥ 关系，但取整符号 ⌊⋅⌋\lfloor \cdot \rfloor⌊⋅⌋ 会导致微小的误差。严格证明需要考虑取整带来的 1 以内的波动。

但在本题（蚯蚓）中，由于 xxx 和 yyy 通常都是整数，且 x≥yx \ge yx≥y 往往意味着 xxx 可能比 yyy 大很多，或者是紧挨着的同阶数。

即便在极端情况下（例如 x=yx=yx=y），公式变为：

⌊px⌋+q≥⌊px+pq⌋\lfloor px \rfloor + q \ge \lfloor px + pq \rfloor⌊px⌋+q≥⌊px+pq⌋

由于 0<p<10 < p < 10<p<1，所以 pq<qpq < qpq<q。

一个数加上 qqq 后取整，通常会大于等于这个数加上一个比 qqq 小的数 pqpqpq 后取整。

只有当 qqq 非常小且 ppp 接近 1 时才需要极度小心的讨论，但在整数域和题目约束下，上述不等式恒成立。

四、 总结

你提到了数学归纳法，这正是这道题解法成立的基石：

  1. 基础步 (Base Case)：初始状态下，原始数组 Q1Q_1Q1​ 是有序的（我们排了序）。
  2. 归纳步 (Inductive Step)：
    • 假设在 ttt 时刻，Q1,Q2,Q3Q_1, Q_2, Q_3Q1​,Q2​,Q3​ 都是有序的。
    • 我们证明了：在 ttt 时刻从任意队列头取出的最大值 xxx，经过切割运算后，生成的新值 xnewx_{new}xnew​，一定会小于等于之前时刻放入 Q2/Q3Q_2/Q_3Q2​/Q3​ 的队尾元素（因为之前放入的元素是由更大的“祖先”生成的，且经历了更长时间的 +q+q+q）。
    • 同时，这一刻生成的 xnewx_{new}xnew​ 会大于等于未来 t+1t+1t+1 时刻生成的 ynewy_{new}ynew​（刚刚证明的单调性）。
    • 因此，Q2Q_2Q2​ 和 Q3Q_3Q3​ 始终保持单调递减。

这就是为什么我们可以把优先队列 O(mlog⁡m)O(m \log m)O(mlogm) 降维打击成 O(m)O(m)O(m) 的根本原因——我们利用数学性质构造了天然的顺序，从而省略了排序的过程。
