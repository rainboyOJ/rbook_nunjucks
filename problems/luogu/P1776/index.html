题目解析

这是一道典型的多重背包问题。

问题描述：给定 nnn 种物品，每种物品有价值 viv_ivi​、重量 wiw_iwi​ 和数量 mim_imi​。有一个容量为 WWW 的背包，要求选择物品装入背包，使得在总重量不超过 WWW 的前提下，总价值最大。

朴素的动态规划做法是三重循环：枚举物品种类 iii，枚举背包容量 jjj，枚举当前物品选择的数量 kkk。状态转移方程为：

dp[j]=max⁡(dp[j],dp[j−k⋅wi]+k⋅vi)dp[j] = \max(dp[j], dp[j - k \cdot w_i] + k \cdot v_i)dp[j]=max(dp[j],dp[j−k⋅wi​]+k⋅vi​)，其中 1≤i≤n1 \le i \le n1≤i≤n，0≤j≤W0 \le j \le W0≤j≤W，0≤k≤mi0 \le k \le m_i0≤k≤mi​ 且 k⋅wi≤jk \cdot w_i \le jk⋅wi​≤j。

这种做法的时间复杂度是 O(W×∑mi)O(W \times \sum m_i)O(W×∑mi​)。根据题目数据范围，W≤4×104W \le 4 \times 10^4W≤4×104，∑mi≤105\sum m_i \le 10^5∑mi​≤105，最坏情况下计算量会达到 4×1094 \times 10^94×109 级别，超时无疑。因此我们需要进行优化。

下面分别介绍两种常见的优化方法：二进制拆分优化和单调队列优化。

方法一：二进制拆分优化 (Binary Splitting Optimization)

1. 原理解析

多重背包问题的瓶颈在于物品数量的限制。如果我们能把第 iii 种物品的 mim_imi​ 件拆分成若干组，每一组看作是一个全新的“0/1 背包”物品（只能选或不选），那么问题就转化为了 0/1 背包问题。

如何拆分最高效？利用二进制的思想。任何一个正整数 mmm 都可以表示为若干个 222 的幂次方的和。我们可以将 mim_imi​ 件物品拆分成数量分别为 1,2,4,8,…,2k,R1, 2, 4, 8, \dots, 2^k, R1,2,4,8,…,2k,R 的若干组，其中 2k≤mi2^k \le m_i2k≤mi​ 且 1+2+4+⋯+2k+R=mi1+2+4+\dots+2^k+R = m_i1+2+4+⋯+2k+R=mi​，RRR 是剩下的余数（可能为 0）。

例如，若某物品有 13 件，我们可以拆分成数量为 1, 2, 4, 6 的四组新物品。

  • 第 1 组：重量 1⋅wi1 \cdot w_i1⋅wi​，价值 1⋅vi1 \cdot v_i1⋅vi​
  • 第 2 组：重量 2⋅wi2 \cdot w_i2⋅wi​，价值 2⋅vi2 \cdot v_i2⋅vi​
  • 第 3 组：重量 4⋅wi4 \cdot w_i4⋅wi​，价值 4⋅vi4 \cdot v_i4⋅vi​
  • 第 4 组：重量 6⋅wi6 \cdot w_i6⋅wi​，价值 6⋅vi6 \cdot v_i6⋅vi​

通过组合这四组新物品（选或不选），我们可以凑出 000 到 131313 之间任意数量的原物品。

2. 复杂度分析

对于每种物品 mim_imi​，拆分后的新物品数量大约是 log⁡2(mi)\log_2(m_i)log2​(mi​) 个。总的新物品数量 N′N'N′ 大约是 ∑log⁡2(mi)\sum \log_2(m_i)∑log2​(mi​)。在最坏情况下（例如 NNN 较小而 mim_imi​ 很大），N′N'N′ 的上界大约是 N×log⁡(max⁡mi)N \times \log(\max m_i)N×log(maxmi​)。

对于题目数据，∑mi≤105\sum m_i \le 10^5∑mi​≤105，拆分出来的新物品总数通常不会超过 10510^5105（实际上远小于这个数，大约在 N×17N \times 17N×17 左右）。

转化后的 0/1 背包时间复杂度为 O(W×N′)O(W \times N')O(W×N′)。这个复杂度通常足以通过此题。

        /**
 * Author by Rainboy blog: https://rainboylv.com github : https://github.com/rainboylvx
 * rbook: -> https://rbook.roj.ac.cn  https://rbook2.roj.ac.cn
 * date: 2026-01-05 08:48:02
 */
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

// 定义新物品结构体
struct Goods {
    int v, w;
};

int n, W;
// dp数组，大小需要超过最大载重
int dp[40005];
vector<Goods> goods_list;

int main() {
    // 加速输入输出
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n >> W;

    // 1. 二进制拆分过程
    for (int i = 0; i < n; ++i) {
        int v, w, m;
        cin >> v >> w >> m;
        // 将m拆分为 1, 2, 4, ...
        for (int k = 1; k <= m; k <<= 1) {
            goods_list.push_back({k * v, k * w});
            m -= k;
        }
        // 处理剩下的余数 R
        if (m > 0) {
            goods_list.push_back({m * v, m * w});
        }
    }

    // 2. 0/1 背包过程
    for (const auto& g : goods_list) {
        // 0/1 背包内层循环倒序
        for (int j = W; j >= g.w; --j) {
            dp[j] = max(dp[j], dp[j - g.w] + g.v);
        }
    }

    cout << dp[W] << endl;

    return 0;
}

方法二：单调队列优化 (Monotonic Queue Optimization)

1. 原理解析

让我们回到朴素的状态转移方程：

dp[j]=max⁡0≤k≤mi{dpprev[j−k⋅wi]+k⋅vi}dp[j] = \max_{0 \le k \le m_i} \{ dp_{prev}[j - k \cdot w_i] + k \cdot v_i \}dp[j]=max0≤k≤mi​​{dpprev​[j−k⋅wi​]+k⋅vi​}

观察这个方程，当我们在计算 dp[j],dp[j+wi],dp[j+2wi],…dp[j], dp[j+w_i], dp[j+2w_i], \dotsdp[j],dp[j+wi​],dp[j+2wi​],… 时，我们会发现它们依赖的状态在不断“滑动”。

我们可以按照容量 jjj 对 wiw_iwi​ 取模的余数 rrr (0≤r<wi0 \le r < w_i0≤r<wi​) 来进行分组。对于固定的余数 rrr，容量 jjj 可以表示为 j=q⋅wi+rj = q \cdot w_i + rj=q⋅wi​+r，其中 qqq 是商。

我们将方程中的 jjj 替换掉，并将上一层状态记为 dppredp_{pre}dppre​：

dp[q⋅wi+r]=max⁡0≤k≤mi{dppre[(q−k)⋅wi+r]+k⋅vi}dp[q \cdot w_i + r] = \max_{0 \le k \le m_i} \{ dp_{pre}[(q - k) \cdot w_i + r] + k \cdot v_i \}dp[q⋅wi​+r]=max0≤k≤mi​​{dppre​[(q−k)⋅wi​+r]+k⋅vi​}

令 k′=q−kk' = q - kk′=q−k，则 k=q−k′k = q - k'k=q−k′。当 kkk 的范围是 [0,mi][0, m_i][0,mi​] 时，k′k'k′ 的范围是 [q−mi,q][q - m_i, q][q−mi​,q]。方程变换为：

dp[q⋅wi+r]=max⁡q−mi≤k′≤q{dppre[k′⋅wi+r]+(q−k′)⋅vi}dp[q \cdot w_i + r] = \max_{q - m_i \le k' \le q} \{ dp_{pre}[k' \cdot w_i + r] + (q - k') \cdot v_i \}dp[q⋅wi​+r]=maxq−mi​≤k′≤q​{dppre​[k′⋅wi​+r]+(q−k′)⋅vi​}

dp[q⋅wi+r]=max⁡q−mi≤k′≤q{dppre[k′⋅wi+r]−k′⋅vi}+q⋅vidp[q \cdot w_i + r] = \max_{q - m_i \le k' \le q} \{ dp_{pre}[k' \cdot w_i + r] - k' \cdot v_i \} + q \cdot v_idp[q⋅wi​+r]=maxq−mi​≤k′≤q​{dppre​[k′⋅wi​+r]−k′⋅vi​}+q⋅vi​

现在核心看大括号里的部分。对于一个固定的余数 rrr，随着商 qqq 的增加，我们需要在范围 [q−mi,q][q - m_i, q][q−mi​,q] 内找到一个 k′k'k′，使得 dppre[k′⋅wi+r]−k′⋅vidp_{pre}[k' \cdot w_i + r] - k' \cdot v_idppre​[k′⋅wi​+r]−k′⋅vi​ 的值最大。这正是一个经典的滑动窗口最大值问题，可以使用单调队列在 O(1)O(1)O(1) 的均摊时间内解决。

我们需要维护一个单调递减队列，队列中存储的是商的下标 k′k'k′。队列里的元素对应的值（即 dppre[k′⋅wi+r]−k′⋅vidp_{pre}[k' \cdot w_i + r] - k' \cdot v_idppre​[k′⋅wi​+r]−k′⋅vi​）是单调递减的。

2. 复杂度分析

  • 外层循环枚举物品种类：NNN 次。

  • 中层循环枚举余数：wiw_iwi​ 次。

  • 内层循环枚举商：W/wiW / w_iW/wi​ 次。

  • 单调队列操作是均摊 O(1)O(1)O(1) 的。

    总时间复杂度 = ∑i=1N(wi×Wwi)=∑i=1NW=O(N×W)\sum_{i=1}^N (w_i \times \frac{W}{w_i}) = \sum_{i=1}^N W = O(N \times W)∑i=1N​(wi​×wi​W​)=∑i=1N​W=O(N×W)。

    代入数据 100×4×104=4×106100 \times 4 \times 10^4 = 4 \times 10^6100×4×104=4×106，这个复杂度非常优秀，是多重背包问题的最优解法之一。

        #include <iostream>
#include <vector>
#include <deque>
#include <algorithm>

using namespace std;

int n, W;
int v[105], w[105], m[105];
// dp[j] 表示容量为 j 时的最大价值
int dp[40005];
// pre_dp 用于存储上一轮物品处理后的状态，用于辅助单调队列计算
int pre_dp[40005];

// 计算单调队列中用于比较的值
inline int calc_val(int idx, int weight, int value, int r) {
    return pre_dp[idx * weight + r] - idx * value;
}

int main() {
    // 加速输入输出
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n >> W;
    for (int i = 1; i <= n; ++i) {
        cin >> v[i] >> w[i] >> m[i];
    }

    // 初始化 dp 数组为 0
    fill(dp, dp + W + 1, 0);

    // 枚举每一种物品
    for (int i = 1; i <= n; ++i) {
        // 备份上一轮的状态
        copy(dp, dp + W + 1, pre_dp);
        // 枚举余数 r
        for (int r = 0; r < w[i]; ++r) {
            deque<int> dq; // 单调队列存储的是商的下标 q
            // 枚举商 q，对应的容量是 j = q * w[i] + r
            for (int q = 0; q * w[i] + r <= W; ++q) {
                int current_capacity = q * w[i] + r;
                int current_val_for_compare = calc_val(q, w[i], v[i], r);

                // 1. 入队操作：维护队列单调递减性
                // 如果队尾元素对应的值小于等于当前值，则队尾元素永远不会成为最大值，弹出
                while (!dq.empty() && calc_val(dq.back(), w[i], v[i], r) <= current_val_for_compare) {
                    dq.pop_back();
                }
                dq.push_back(q);

                // 2. 出队操作：检查队头是否滑出窗口范围
                // 窗口大小为 m[i]+1，有效范围是 [q - m[i], q]
                if (dq.front() < q - m[i]) {
                    dq.pop_front();
                }

                // 3. 更新 DP 状态
                // 队头保存的是窗口内的最优 k'
                int best_k_prime = dq.front();
                // 代入公式：dp[j] = (pre_dp[k'*w+r] - k'*v) + q*v
                // 化简后为：dp[j] = pre_dp[k'*w+r] + (q - k') * v
                // 其中 (q - k') 就是选择当前物品的数量
                dp[current_capacity] = pre_dp[best_k_prime * w[i] + r] + (q - best_k_prime) * v[i];
            }
        }
    }

    cout << dp[W] << endl;

    return 0;
}

总结

对于多重背包问题：

  1. 数据范围较小时，可以直接用朴素的三重循环 O(W∑mi)O(W \sum m_i)O(W∑mi​)。
  2. 一般数据范围（如本题，WWW 中等，∑mi\sum m_i∑mi​ 较大），推荐使用二进制拆分优化，转化为 0/1 背包，复杂度 O(W∑log⁡mi)O(W \sum \log m_i)O(W∑logmi​)，实现简单，不易出错。
  3. 对时间要求极高或数据范围更大时，使用单调队列优化，复杂度 O(NW)O(NW)O(NW)，是理论上的最优复杂度，但实现细节较多，推导公式需要仔细。
